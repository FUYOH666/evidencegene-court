# Staff review — EvidenceGene Court v0.2 (`v0.1.0..HEAD`)

Reviewer: staff-level pipeline (full mode). Date: 2026-06-13.

## Scope

- **Goal of the change:** add three adversarial layers (injection harness,
  counterfactual ablation, jury of models) plus a kill-chain timeline and HTML
  report, without weakening the v0.1 architectural guarantees.
- **Modules touched:** `court/binding.py` (new), `court/jury.py` (new),
  `analysis/ablation.py` (new), `redteam/*` (new), `report/*` (new),
  `fixture.py` (new), `court/orchestrator.py` (refactor), `artifacts/store.py`,
  `attestation/{models,serializer}.py`, `court/{llm,cli}.py`, `config.py`,
  plus tests and docs.
- **Excluded:** generated artifacts (`reports/`), `evidencegene-demo.mp4`,
  `uv.lock`, rendered `report.html`/`REDTEAM_REPORT.md`.

## Correctness & logic

- `Court` refactor into `collect()` + `trial(model=...)` preserves the original
  `investigate()` behavior; `trial` takes a local copy of `summaries`
  (`summaries = list(summaries)`), so jurors do not interfere. Good.
- `bind_claim` is pure and now the single source of truth for refs/tier; both
  the orchestrator and ablation call it, removing duplication. Good.
- Jury vote counting uses a set of juror models per entity (one vote per juror),
  correct even if an entity appears in several dispositions.
- Minor: during a jury run, each juror's `trial` may re-run arbiter-requested
  follow-up tools, appending duplicate artifacts to the shared store. Not
  incorrect (binding dedups by entity; audit chain stays valid) but slightly
  wasteful. Acceptable for the hackathon scope.

## Architecture fit

- New modules respect the layering: tools stay read-only and typed; nothing new
  is added to the MCP surface; publication still flows only through
  `FindingSerializer`. The red-team harness attacks the boundaries rather than
  bypassing them. Strong fit.

## Security & privacy

- No secrets, IPs, or absolute user paths in tracked files (`.gitignore` ignores
  `.env`, `*.key`, evidence, and all of `reports/`).
- Red-team `rt-06` asserts the MCP surface has no destructive verb tools — a
  standing regression guard for the core invariant.
- Local-first: jury and court run against a configurable OpenAI-compatible
  endpoint; evidence never leaves the host by default.

## Reliability & ops

- Jury is resilient: a juror raising `LLMError` abstains and is logged as a
  `jury_ballot` with `abstained: true`, rather than crashing the panel.
- Logging via `logging` (not `print`) in library code. CLI uses `print` for
  user-facing scorecards, which is appropriate.
- `egc-court health` covers LLM endpoint, tools, and audit-chain status.

## Performance

- All new analysis (binding, ablation, redteam, timeline, report) is in-memory
  over already-collected artifacts; negligible cost. The only LLM cost is the
  jury (N court runs), which is opt-in and documented.

## Tests & verification

- 20 tests, all offline (jury uses a `FakeChatClient`); `ruff` clean; CI green.
- Covers happy and negative paths: serializer rejection, tier downgrade, chain
  tamper detection, ablation collapse, jury consensus/minority drop, report
  sections. No disabled tests or blanket ignores. Meets the repo Definition of
  Done (tests + README + CHANGELOG).

## Docs & DX

- README has a v0.2 section; CHANGELOG `[0.2.0]`; ATLAS mapping and accuracy
  report updated. `TRY_IT_OUT.md` correctly requires `uv sync --extra dev
  --extra forensics` (both occurrences), which prevents the `vol`-missing
  pitfall.
- Stale demo link in `README.md` (old YouTube id) — fixed in this round.

## Verdict

**Ready to submit after F1** (a small, low-risk lazy-init fix) and the demo-link
update. No blockers. Architecture, security, tests, and docs are in strong
shape; the adversarial layers materially raise the submission's value against
the judging criteria (architectural guardrails, tested-for-bypass, audit trail).
